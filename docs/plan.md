# plan.md

## Цель проекта
Создать персональный сайт-платформу для актрисы, певицы, ведущей, voice performer и мультиязычной артистки.

Сайт должен работать как:

- casting profile;
- артистическое портфолио;
- voice / dubbing portfolio;
- event host portfolio;
- press kit;
- центральная точка доверия из Google, Instagram, TikTok, YouTube, Spotlight, IMDb, кастинг-платформ и личных рекомендаций.

## Главная продуктовая гипотеза
Сайт будет эффективен, если он не просто показывает «много талантов», а превращает их в понятное профессиональное предложение:

> актриса + языки + голос + музыка + сцена + live events.

## MVP
MVP должен быть маленьким, быстрым и сильным.

### Обязательные страницы / секции
1. Home.
2. Acting / Showreel.
3. Voice / Dubbing.
4. Languages.
5. Music / Vocal / Piano.
6. Hosting / Events.
7. Casting Profile.
8. Contact.
9. Download CV / Press Kit.

### Обязательные материалы
- 1 главный headshot;
- 4–8 профессиональных фото;
- acting showreel 60–90 секунд;
- voice samples минимум на 3 языках;
- короткие language samples;
- vocal video или audio;
- piano video или audio;
- hosting photos/video, если есть;
- PDF CV;
- короткая bio на английском;
- короткая bio на словацком;
- контактный email.

## Фазы работ

### Phase 0 — Strategy / Positioning
Цель: понять, что сайт продаёт и кому.

Задачи:

- утвердить позиционирование;
- определить основную аудиторию;
- собрать список работ / credits;
- выбрать имя домена;
- определить языки первой версии;
- решить, будет ли сайт публично индексироваться полностью или часть будет private.

Deliverables:

- brand_positioning.md;
- sitemap draft;
- список нужных материалов;
- финальная формула позиционирования.

### Phase 1 — Content Inventory
Цель: собрать все материалы до дизайна.

Задачи:

- собрать фото;
- собрать видео;
- собрать аудио;
- подготовить CV;
- выписать theatre / film / ads / series credits;
- выписать языки и уровень;
- выписать special skills;
- понять, какие материалы нужно дозаписать.

Deliverables:

- media_inventory.md;
- content_model.md;
- список missing assets.

### Phase 2 — Information Architecture
Цель: сделать структуру, где всё быстро находится.

Задачи:

- финализировать sitemap;
- определить URL;
- продумать навигацию;
- продумать разные пользовательские сценарии;
- определить, что выводится на главной, а что на внутренних страницах.

Deliverables:

- architecture.md;
- user_flows.md;
- page briefs для каждой страницы.

### Phase 3 — Visual Direction
Цель: определить внешний стиль без перегруза.

Задачи:

- выбрать mood: cinematic / elegant / minimal / international;
- выбрать типографику;
- выбрать цветовую палитру;
- определить стиль фото и превью;
- создать wireframes;
- создать дизайн главной и casting profile.

Deliverables:

- design_system.md;
- дизайн-макеты;
- component list.

### Phase 4 — Development MVP
Цель: собрать первую рабочую версию.

Задачи:

- создать frontend structure;
- подключить контент;
- реализовать мультиязычность;
- реализовать медиа-компоненты;
- реализовать download links;
- реализовать contact CTA;
- настроить SEO базу;
- настроить аналитику.

Deliverables:

- staging website;
- базовая админка / content editing flow;
- SEO metadata;
- analytics events.

### Phase 5 — Media Optimization
Цель: сделать сайт быстрым и приятным.

Задачи:

- оптимизировать изображения;
- вынести видео на подходящую платформу;
- сделать lazy loading;
- добавить poster images;
- проверить mobile playback;
- проверить Core Web Vitals.

Deliverables:

- optimized image set;
- video embed strategy;
- performance report.

### Phase 6 — QA / Launch
Цель: выпустить сайт без стыда.

Задачи:

- проверить тексты;
- проверить все языки;
- проверить формы и email;
- проверить ссылки на соцсети;
- проверить mobile / tablet / desktop;
- проверить SEO / Open Graph;
- подключить домен;
- открыть индексацию;
- отправить сайт в Google Search Console.

Deliverables:

- production release;
- launch checklist;
- post-launch backlog.

### Phase 7 — Growth
Цель: превратить сайт в рабочий инструмент.

Задачи:

- добавить private casting links;
- добавить новые showreels;
- добавить press kit;
- добавить case pages по сильным проектам;
- добавить страницы под voice / hosting / languages;
- отслеживать, что реально открывают клиенты.

Deliverables:

- monthly content updates;
- analytics review;
- backlog improvements.

## Рекомендуемый порядок запуска
Не делать сразу идеальную систему.

1. MVP на 1–2 языках.
2. Acting + Voice + Contact как ядро.
3. Music / Hosting / Skills как усилители.
4. Private links и advanced media позже.
5. Полная мультиязычность после подтверждения структуры.

## Что не входит в MVP
- сложная CMS, если контент редко меняется;
- блог;
- магазин;
- пользовательские аккаунты;
- heavy 3D;
- многоуровневая админка;
- автоматический booking calendar;
- автоплей аудио;
- длинные статьи о творческом пути.

## Метрики успеха
### Количественные
- рост прямых переходов по имени;
- клики по Contact;
- скачивания CV;
- просмотры showreel;
- прослушивания voice samples;
- переходы из Instagram / TikTok / casting platforms;
- низкий bounce rate на casting profile.

### Качественные
- легче отправлять себя на кастинги;
- меньше объяснений в переписке;
- клиенты быстрее понимают диапазон навыков;
- сайт выглядит профессиональнее, чем соцсети;
- сайт можно отправить агенту / режиссёру без дополнительного контекста.

## Риски
1. Слишком много талантов без иерархии.
2. Сайт выглядит как дизайнерский эксперимент, а не как рабочий инструмент.
3. Нет сильного showreel.
4. Слабые аудио samples.
5. Медленная загрузка из-за видео и фото.
6. Мультиязычность растягивает запуск.
7. Биография слишком длинная и общая.

## Антириски
- сначала контент, потом дизайн;
- главная роль — актриса;
- языки и voice — как сильное отличие;
- короткие доказательства вместо длинных описаний;
- кастинг-страница без лишней красоты;
- регулярное обновление материалов.

### Phase 4.5 — Creative Tech Prototype
Цель: проверить вау-слой отдельно от основного сайта, чтобы не сломать MVP.

Задачи:

- выбрать 1–2 hero concepts;
- сделать static fallback;
- собрать WebGL/R3F prototype;
- проверить WebGPU только как optional path;
- подключить GSAP/ScrollTrigger для одной cinematic sequence;
- проверить mobile performance;
- проверить reduced-motion;
- решить, что идёт в production, а что остаётся в lab.

Deliverables:

- creative prototype;
- performance notes;
- fallback design;
- feature flags;
- решение go/no-go для каждого эффекта.

### Phase 5.5 — Extreme Performance Pass
Цель: удержать вау-слой без ощущения тяжёлого сайта.

Задачи:

- lazy-load creative chunks;
- оптимизировать 3D assets;
- внедрить DPR cap;
- добавить FPS/quality monitor;
- вынести тяжёлые расчёты в workers;
- подготовить KTX2/Basis textures, если нужны 3D textures;
- проверить Meshopt/Draco для GLB;
- проверить mobile и Safari;
- отключить canvas на `/casting` по умолчанию.

Deliverables:

- performance budget report;
- optimized asset pack;
- device QA notes;
- final creative feature flags.
