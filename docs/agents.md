# agents.md

## Роль документа
Этот файл задаёт правила работы для AI-агентов, разработчиков и контент-редакторов, которые помогают создавать персональный сайт актрисы / performer.

Главная цель проекта: сделать не «красивую визитку», а профессиональную платформу, которая быстро продаёт артистку кастинг-директорам, агентам, режиссёрам, студиям дубляжа, event-агентствам и коммерческим клиентам.

## Северная звезда проекта
Сайт должен за 20–40 секунд отвечать на вопросы:

1. Кто она?
2. Как выглядит?
3. Как звучит?
4. На каких языках работает?
5. Что уже делала?
6. Чем отличается от других актрис?
7. Как быстро с ней связаться?

Если любая функция, анимация, текст или компонент не помогает ответить на эти вопросы — он должен быть упрощён, перенесён ниже или удалён.

## Позиционирование
Базовая формула:

> Multilingual actress and performer for screen, stage, voice and live events.

Русская версия:

> Мультиязычная актриса и performer для кино, сцены, голоса и live-мероприятий.

Не позиционировать героиню как «человека, который умеет всё подряд». Позиционировать как актрису с редкой комбинацией навыков: языки, голос, музыка, сцена, ведущая, physical skills.

## Приоритеты аудитории
### Primary audience
- кастинг-директора;
- режиссёры;
- агенты;
- продюсеры;
- рекламные агентства;
- студии дубляжа;
- event-агентства.

### Secondary audience
- журналисты;
- организаторы культурных событий;
- фотографы / режиссёры клипов;
- потенциальные коллаборации;
- зрители, которые нашли артистку через соцсети.

## Тон сайта
Сайт должен ощущаться:

- профессионально;
- дорого;
- спокойно;
- международно;
- артистично, но не хаотично;
- быстро и удобно.

Не использовать тон «я творческая душа, ищу себя». Использовать тон «я профессиональная артистка, с которой удобно работать».

## Главные UX-правила
1. Первый экран должен показать имя, профессию, фото и ключевые CTA.
2. Контакты должны быть доступны с любого места сайта.
3. Showreel, voice samples и CV должны открываться максимально быстро.
4. Видео и аудио не должны автозапускаться со звуком.
5. Не прятать важную информацию в сложных анимациях.
6. Не делать креативность ценой удобства.
7. Мобильная версия важнее десктопной: кастинг-директор часто открывает ссылку с телефона.
8. Все ключевые материалы должны быть доступны за 1–2 клика.

## Контентная иерархия
Порядок важности:

1. Acting / screen / stage.
2. Languages.
3. Voice / dubbing.
4. Music / vocal / piano.
5. Hosting / events.
6. Special skills: aerial gymnastics, drawing, translation.
7. Biography / personal story.

Биография не должна доминировать. Визуальные и голосовые доказательства важнее длинного текста.

## Техническая философия
Проект может быть технически сильным, но не должен казаться тяжёлым.

Рекомендованный подход:

- статическая или почти статическая генерация;
- высокая скорость загрузки;
- строгая оптимизация медиа;
- мультиязычная структура;
- чистые URL;
- простое редактирование контента;
- SEO по имени, профессии, языкам и регионам;
- private links для кастинговых материалов.

## Предпочтительный стек, если нет другого решения
Мнение по умолчанию:

- Astro или Next.js как frontend;
- TypeScript;
- MDX / JSON / headless CMS для контента;
- Tailwind CSS или CSS Modules;
- Vimeo / YouTube unlisted / Bunny Stream / Cloudflare Stream для видео;
- локальные optimized images через build pipeline;
- Plausible / Umami / Google Analytics для аналитики;
- Vercel / Netlify / Cloudflare Pages для деплоя.

Не привязываться к стеку фанатично. Главные требования: скорость, простота поддержки, мультиязычность, хорошая работа с медиа.

## Структура сайта
Минимальный production scope:

- Home;
- Acting;
- Voice;
- Music;
- Hosting;
- Skills;
- Gallery;
- Casting Profile;
- Contact;
- Press Kit / Downloads.

Можно реализовать как один длинный сайт с якорями + отдельная практичная страница `/casting`.

## Правила для AI-генерации текстов
AI может помогать с текстами, но итог должен звучать человечески и профессионально.

Запрещено:

- писать слишком пафосно;
- использовать клише «passionate artist», «creative soul», «unique journey» без конкретики;
- раздувать опыт, которого нет;
- делать неподтверждённые claims;
- использовать фразы, которые сложно доказать: «одна из лучших», «ведущая актриса региона».

Разрешено:

- усиливать формулировки;
- структурировать опыт;
- делать короткие bio в разных длинах;
- адаптировать тексты под английский / словацкий / польский / итальянский / украинский / русский;
- писать разные версии под кастинг, voice, event и press.

## Правила для медиа
1. Все изображения должны иметь alt-тексты.
2. Headshots и production stills должны быть разделены.
3. Видео должно иметь постеры / превью.
4. Voice samples должны иметь названия, язык, жанр и длительность.
5. Не смешивать бытовые фото с профессиональными в одном блоке.
6. Не использовать слишком много материалов: лучше 6 сильных фото, чем 40 средних.

## Правила для мультиязычности
Основные языки сайта на старте:

- English;
- Slovak.

Далее можно добавить:

- Polish;
- Italian;
- Ukrainian;
- Russian.

Венгерский указывать как базовый / elementary / conversational только если это честно соответствует уровню.

Не переводить все страницы механически. Можно начать с полного английского сайта + коротких локализованных landing-блоков и voice/language samples.

## SEO-правила
Приоритет SEO:

1. Имя + фамилия.
2. Actress + город / страна.
3. Multilingual actress.
4. Slovak / Polish / Italian / Ukrainian / Russian / English actress.
5. Voice actress / dubbing actress.
6. Event host / presenter.

Каждая страница должна иметь:

- title;
- meta description;
- canonical URL;
- hreflang для языковых версий;
- Open Graph image;
- структурированные данные Person / PerformingGroup / WebSite там, где уместно.

## Privacy & safety
Не публиковать:

- домашний адрес;
- личные документы;
- личные номера, если предпочтительнее рабочий email;
- неотобранные private casting videos без защиты;
- материалы с чужими правами без разрешения.

Для контакта лучше использовать профессиональный email на домене сайта.

## Definition of Done
Страница считается готовой, если:

- она быстро открывается на мобильном;
- есть понятный CTA;
- контент не выглядит пустым;
- все медиа оптимизированы;
- тексты вычитаны;
- есть alt-тексты;
- работает мультиязычная навигация;
- форма / email / ссылки проверены;
- страница индексируется, если она публичная;
- страница закрыта от индексации, если она private.

## Главный принцип
Сайт должен не демонстрировать возможности разработчика, а увеличивать стоимость артистки в глазах клиента.

## Creative Tech Layer rules
Технический вау-слой разрешён и желателен, но только как усилитель бренда, а не как главный смысл сайта.

Правила:

1. Core content должен работать без WebGPU/WebGL.
2. Canvas/3D не должен блокировать hero text, showreel, CV и contact.
3. WebGPU — optional ultra tier, не обязательная зависимость.
4. WebGL 2 / Three.js / R3F — основной production-путь для 3D.
5. GSAP/ScrollTrigger использовать для cinematic storytelling, но не для scroll hijacking.
6. Framer Motion / Motion One использовать для мягких UI transitions.
7. Все тяжёлые эффекты должны иметь fallback.
8. Все тяжёлые эффекты должны иметь reduced-motion вариант.
9. 3D, shaders, physics, WASM и workers добавлять только если они усиливают артистку.
10. Casting Profile должен оставаться быстрым и почти без декоративного слоя.

Запрещено:

- строить весь сайт как canvas-only experience;
- прятать контакты и CV за анимациями;
- включать звук без user action;
- грузить WebGPU/WebGL chunks в critical path;
- добавлять физику/particles только ради демонстрации кода;
- делать mobile-версию хуже desktop-версии.

Разрешено и желательно:

- Stage Light hero scene;
- subtle shader portrait;
- multilingual voice galaxy;
- audio waveform visualization;
- cinematic scroll sections;
- private casting mode без вау-слоя;
- feature flags для creative modules.
